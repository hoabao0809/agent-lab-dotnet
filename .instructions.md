# Workspace Instructions

## Pre-Commit Checklist

Before pushing code, run these mandatory checks:

- [ ] **Lint:** `dotnet format --verify-no-changes` (or fix with `dotnet format`)
- [ ] **Build:** `dotnet build` (no errors)
- [ ] **Test:** `dotnet test` (all tests pass)

---

## Quick Start

**Soc Ops** is a Blazor WebAssembly app (.NET 10.0, C#) demonstrating GitHub Copilot agents with a Bingo quiz game.

```bash
cd SocOps
dotnet restore
dotnet build
dotnet run  # Launches at http://localhost:5166
```

---

## Project Structure

```
SocOps/
├── Components/    # Razor UI components (BingoBoard, GameScreen, etc.)
├── Layout/        # MainLayout, NavMenu
├── Pages/         # Home, Counter, Weather, NotFound
├── Models/        # BingoLine, BingoSquareData, GameState
├── Services/      # BingoGameService, BingoLogicService
├── Data/          # Questions.cs
└── wwwroot/       # Static assets
```

---

## Development

**Make Changes:**
- Edit `.razor` files (UI) or `.cs` files (logic)
- Auto-reload with `dotnet run`

**Add Components:**
- Create in `Components/` or `Pages/`
- Register in `App.razor` if needed

**Modify Game Logic:**
- `Services/BingoGameService.cs` - Game mechanics
- `Services/BingoLogicService.cs` - Validation
- `Data/Questions.cs` - Quiz questions

---

## Commands

```bash
dotnet restore    # Install dependencies
dotnet build      # Compile
dotnet run        # Start dev server
dotnet clean      # Remove artifacts
dotnet test       # Run tests
```

---

## Troubleshooting

**Port in use:** `dotnet run --urls "http://localhost:5167"`

**Build issues:** `dotnet clean && dotnet restore && dotnet build`
